{
  "title_id": "PPSA01411",
  "name":     "Game Name (optional)",
  "cheats": [

      "name": "Infinite Health",
      "mods": [
        { "offset": "0x12340", "on": "9090", "off": "0102", "absolute": false }
      ]
    }
  ]
}
Offsets default to relative to the eboot image base; set "absolute": true for raw addresses. Hex strings may use spaces, dashes, or colons as separators.

Master on/off for the bundled ftpsrv. Off cleanly TERMs the daemon (5 s grace then SIGKILL); On respawns it on the configured port.
Default 2121. Saving a new port automatically kills the running daemon and respawns it on the new port — no need to re-send Sonic Loader. Resend payload kills + respawns the daemon on the current port without changing it.
Default anonymous — open access on the LAN. Switch to Username + password to require credentials. Auth is plaintext (FTP itself is plaintext); fine for "stop the kid next door from poking around" but don't reuse a real password.
Auto lets the client pick (sends TYPE I for binary or TYPE A for ASCII). Binary / ASCII force the per-session default server-side regardless of what the client sends — useful when a client keeps defaulting to ASCII and corrupting binary uploads with CRLF rewrites.
🌡 Fan control /dev/icc_fan threshold (pinned)

Sets the fan threshold via /dev/icc_fan ioctl (0xC01C8F07). Same approach used by Scene-Collective/ps4-fan-threshold. Lower temperatures spin the fan up sooner.

Range 30–90°C. The PS5 clamps to its supported range internally.
Drag points or edit values. Persistence-only in this build — the single threshold above is what the firmware actually applies. A future watcher could evaluate this curve to switch thresholds based on measured temp.

🪪 Offline account activation ps5-payload-dev/offact (registry only)

Lists every user slot stored in the system registry on this console (up to 16). For each slot you can: Auto-activate (writes a synthetic id derived from the slot name + type=np + flags=0x1002), Edit ID (type a custom ID, hit Save and the slot is fully activated in one shot), or Clear the activation. None of this requires signing into the user — Sonic Loader writes the registry directly via sceRegMgr*. Logic vendored from ps5-payload-dev/offact.

⚠️ Does not enable Remote Play, save export/import, or PSN online — those need real PSN auth. This is "make the slot look activated to ShellCore" only.
ℹ️ Reboot the console after Save before running NP fake sign-in further down this Settings page. ShellCore caches the per-user account state at boot; the new registry values only take effect after a full reboot, and np-fake-signin reads them at start-up to populate auth.dat / config.dat. 🧄 Garlic Worker & SaveMgr community save processing for garlicsaves.com
🧄 Garlic Worker On
Completely harmless — the worker just polls garlicsaves.com for queued save-decrypt / encrypt / re-region jobs and runs them on this console while it's idle. It does not affect game performance during active use — it's small, single-threaded, sleeps between polls, and yields immediately when the foreground app needs the CPU.

Leaving it on helps the entire community: every active worker cuts the queue at garlicsaves.com so users get their saves back faster, and the impact is dramatic right after a new kernel exploit drops and demand spikes.
